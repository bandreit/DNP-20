API Request Sub-path Description Exceptions
/Families GET
Use this to get a Collection of families.
Example:
  • .../Families
 May return 500 Internal Server Error, if something failed.
/{id:int}
Use this to get a specifc family.
Example:
  • .../Families/1
 May return 500 Internal Server Error, if something failed.
POST
Use this to create a new Family. Provide the Family as json in the request body.
Example:
  • .../Families
 May return 400 Bad Request, if the provided Family is invalid. May return 500 Internal Server Error, if something failed.
PUT
Use this to update a Family. Provide the Family as json in the request body.
Example:
  • .../Families/1
 May return 500 Internal Server Error, if something failed.
DELETE
Use this to delete a Family. The deleted family is returned. You must provide two query parameters:
  • streetname
  • housenumber
Example:
  • .../Families?streetname=farvergade&housenumber=3
 May return 500 Internal Server Error, if something failed.

Adult

The Adult is sent back and forth as json.
The json format is seen in the example below:
{
    "JobTitle": "Tutor", 
    "ID": 0,
    "FirstName": "Aryan",
    "LastName": "Gifford",
    "HairColor": "Black",
    "EyeColor": "Brown",
    "Age": 56,
    "Weight": 99.2,
    "Height": 172,
    "Sex": "M"
}
API Request Description Exceptions
/Adults GET
This request will return a collection of Adults.
Example:
  • .../Adults
 May return 500 Internal Server Error, if something fails.
POST
Use this to create a new Adult. Returns the created Adult. Provide the Adult as json in the request body.
Example:
  • .../Adults

May return 400 Bad Request, if the data is invalid.
May return 500 Internal Server Error, if something fails when creating the adult.

Authenticate

The User is sent back and forth as json.
The json format is seen in the example below:
{
    "City" = "Aarhus",
    "Domain" = "hotmail.com",
    "Password" = "654321",
    "Role" = "Student",
    "BirthYear" = 1998,
    "SecurityLevel" = 2,
    "UserName" = "Jakob"
}
